# AI Usage Widget

Claude(claude.ai), Codex(ChatGPT), Gemini(Google) 사용량을 항상 화면에 띄워두고 보는 Windows 데스크톱 위젯입니다.
코딩하다가 매번 각 서비스 설정 화면을 열어서 확인하는 게 번거로워서 만들었습니다.

- **Claude**: 5시간 세션 한도, 주간 한도, (옵션) Fable 모델 주간 한도
- **Codex**: 5시간 한도, 주간 한도
- **Gemini**: 5시간 한도, 주간 한도
- 트레이 메뉴에서 **서비스별로 개별 on/off** (최소 하나는 항상 켜져 있어야 함)
- **1분마다 자동 새로고침**됩니다
- 화면에 항상 떠 있는 **위젯 카드** 모드와, 작업표시줄 **트레이 아이콘**만 쓰는 모드 중 선택할 수 있습니다
- 위젯 크기(소/중/대), 투명도, 색상 테마를 취향대로 조절할 수 있습니다
- 별도 서버나 API 키 없이, 각 서비스에 로그인한 세션으로 직접 값을 읽어옵니다

## 화면 구성

### 위젯 카드

화면 한쪽에 항상 떠 있는 작은 카드입니다. 여러 서비스를 켜면 구역이 위아래로 나뉘어 표시되고, 하나만 켜면 그 구역만 표시되며 창 크기도 자동으로 조절됩니다.

- Claude 구역: 5시간 세션 / (옵션) Fable / 주간 한도 원형 그래프
- Codex 구역: 5시간 한도 / 주간 한도 원형 그래프
- Gemini 구역: 5시간 한도 / 주간 한도 원형 그래프
- 기본(컬러풀) 테마에서는 원마다 고유색이 있습니다: 5시간=청록, Fable=핑크, 주간=노랑. 튀는 색이 싫으면 트레이 메뉴에서 "차분한 톤(무채색)"으로 바꿀 수 있습니다
- 카드 아무 곳이나 **드래그**해서 원하는 위치로 옮길 수 있고, 위치는 자동 저장되어 다음 실행 때도 유지됩니다
- 우측 상단 새로고침(⟳) 버튼으로 즉시 다시 불러올 수 있습니다
- 로그인이 필요한 서비스는 그 구역에 "로그인 필요 → 클릭" 버튼이 나타납니다

### 트레이 아이콘 모드

위젯 카드 대신 작업표시줄 트레이 아이콘 하나만 쓰고 싶다면 이 모드로 바꿀 수 있습니다. 아이콘에 마우스를 올리면 툴팁으로 켜져 있는 서비스들의 숫자가 모두 보입니다.

## 설치

### 1) 설치 파일로 설치 (추천)

[Releases](../../releases)에서 `AIUsageWidget-Setup-x.x.x.exe`를 받아 실행하면 자동으로 설치되고 바로 실행됩니다.

> Windows 전용입니다 (Mac 미지원).
> 서명되지 않은 설치 파일이라 처음 실행 시 Windows 보안 검사로 잠깐 반응이 없어 보일 수 있습니다 — 강제 종료하지 말고 몇 분 기다려주세요.

### 2) 소스에서 직접 빌드

```bash
git clone <이 저장소 주소>
cd AIUsageWidget
npm install
npm run gen-assets   # 아이콘 생성
npm start            # 개발 모드로 실행
npm run dist          # Windows 설치 파일(.exe) 빌드 → dist/ 폴더
```

Node.js가 설치되어 있어야 합니다.

## 처음 실행 시

기본값은 **Claude만 켜짐**입니다. Codex, Gemini도 보고 싶으면 트레이 우클릭 → 해당 서비스 "표시" 체크.

각 서비스는 처음엔 "로그인이 필요해요"라고 표시됩니다.

1. 작업표시줄 트레이 아이콘을 **우클릭**
2. **"Claude 로그인 창 열기"** / **"Codex 로그인 창 열기"** / **"Gemini 로그인 창 열기"** 중 원하는 것 클릭
3. 뜨는 창에서 평소처럼 로그인 (Google/이메일/SSO 등)
4. 로그인이 완료되면 창이 자동으로 닫히고, 위젯에 실제 사용량이 표시됩니다

로그인 세션은 앱에 저장되기 때문에 이후에는 다시 로그인할 필요가 없습니다.
(단, 해당 서비스에서 로그아웃하거나 세션이 만료되면 다시 로그인해야 합니다.)

## 트레이 메뉴

작업표시줄 숨겨진 아이콘 화살표(`^`)를 눌러 아이콘을 찾은 뒤 **우클릭**하면 다음 메뉴가 나옵니다.

| 메뉴 | 설명 |
|---|---|
| 위젯 카드로 보기 | 화면에 항상 떠 있는 카드 형태로 표시 (라디오 선택) |
| 트레이 아이콘으로만 보기 | 카드를 숨기고 트레이 아이콘 + 툴팁으로만 표시 (라디오 선택) |
| 항상 위로 고정 | 켜면 다른 창들 위에 항상 떠 있고, 끄면 일반 창처럼 다른 창에 가려질 수 있습니다 |
| 위젯 투명도 | 위젯 카드의 불투명도를 100% / 85% / 70% / 55% 중에서 선택 |
| 위젯 크기 | 위젯 카드의 전체 크기를 소(80%) / 중(100%, 기본값) / 대(130%) 중에서 선택 |
| 색상 테마 | 컬러풀(청록/핑크/노랑) 또는 차분한 톤(무채색) 중 선택 |
| 위젯에 Fable 표시 | Claude 구역에 Fable 주간 사용률 원을 추가/제거 (트레이 툴팁에는 이 설정과 무관하게 항상 표시됨) |
| Claude 표시 / Codex 표시 / Gemini 표시 | 각 서비스를 위젯·트레이에 표시할지 독립적으로 선택 (최소 하나는 항상 켜져 있어야 함) |
| 지금 새로고침 | 기다리지 않고 즉시 최신 사용량을 다시 불러옴 |
| Claude / Codex / Gemini 로그인 창 열기 | 해당 서비스 로그인 창을 띄움. 로그인 완료가 감지되면 창이 자동으로 닫힙니다 |
| Windows 시작 시 자동 실행 | 켜두면 컴퓨터 부팅 시 위젯이 자동으로 실행됩니다 |
| 사용법 | 간단한 사용법 안내 창을 엶 |
| 디버그 로그 열기 | 값이 이상하거나 로그인이 안 될 때 확인할 수 있는 로그 파일(`%APPDATA%/AIUsageWidget/debug.log`)을 엶 |
| 종료 | 앱을 완전히 종료 |

트레이 아이콘을 **왼쪽 클릭**하면(위젯 카드 모드일 때만) 카드를 보이기/숨기기 토글할 수 있습니다.

## 어떻게 동작하나요?

세 서비스 모두 사용량을 조회하는 공개 API가 없어서, 로그인된 세션으로 실제 설정 페이지를 숨겨진 창에서 열어 화면에 표시된 텍스트를 읽어와 그래프로 그려줍니다.

- Claude: `claude.ai`의 설정 → 사용량 페이지에서 "현재 세션 N% 사용됨" 같은 문구를 파싱
- Codex: `chatgpt.com`의 Codex 애널리틱스 페이지에서 "5시간 사용 한도 N% 남음" 같은 문구를 파싱 (Codex는 "남음" 기준이라 "사용됨"으로 환산해서 표시)
- Gemini: `gemini.google.com/usage` 페이지에서 "현재 사용량 N% 사용됨" / "주간 한도 N% 사용됨" 문구를 파싱

화면에는 보이지 않는 창에서 조용히 이루어지며, 1분마다 자동으로 반복됩니다.

이 방식의 특성상 각 서비스의 페이지 구성이 바뀌면 값을 못 읽어올 수 있습니다. 그런 경우 이슈로 알려주세요.

## 업데이트

새 버전 설치 파일을 다시 실행하면 기존 설치 위에 덮어써지고, **로그인 세션은 그대로 유지**됩니다. 다시 로그인할 필요 없습니다.

## 문제 해결

- **"로그인이 필요해요"가 계속 뜬다**: 트레이 우클릭 → 해당 서비스 "로그인 창 열기"로 다시 로그인해주세요.
- **숫자가 안 바뀐다 / 이상하다**: 트레이 우클릭 → "지금 새로고침"으로 즉시 갱신해보세요. 그래도 안 되면 "디버그 로그 열기"로 로그를 확인해주세요.
- **트레이 아이콘이 여러 개 뜬다**: 중복 실행 방지 기능이 내장되어 있어 발생하지 않습니다.
- **설치 직후 첫 실행이 몇 분간 반응이 없다**: 서명되지 않은 설치 파일이라 Windows가 처음 실행할 때 보안 검사를 하며 생기는 일회성 지연입니다. 강제 종료하지 말고 잠시 기다려주세요.

## 개인정보

- 로그인 정보(쿠키)는 이 앱의 로컬 프로필 폴더(`%APPDATA%/AIUsageWidget`)에 서비스별로 분리 저장되며 외부로 전송되지 않습니다.
- 각 서비스 서버 외에는 어떤 곳으로도 데이터를 보내지 않습니다.

## 기술 스택

Electron + 순수 HTML/CSS/JS. 별도 프레임워크 없이 가볍게 만들었습니다.
